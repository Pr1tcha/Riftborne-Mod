# Codex MVP Status

Source: `TZ_Karmanny_Kodex_Riftborne.pdf`, version 0.1 dated 2026-06-22.

## Implemented

- Pocket Codex item with persistent per-item field data.
- Right-click Pocket Codex interface.
- Previous screen, next screen, and action key bindings.
- Server-authoritative ray scanning with configurable distance.
- Scanning support for rifts, Rift Splinters, Contour blocks, and both interspaces.
- Short field records with threat level and practical recommendation.
- Queued and damaged record states.
- Codex Dock block with one persistent Pocket Codex slot.
- Physical synchronization requiring an adjacent Codex Laptop.
- Normal records become full laptop archive entries after synchronization.
- Damaged records enter the laptop Decryptor queue.
- Laptop desktop applications for archive, synchronization status, and Decryptor.
- Minimal Decryptor prototype that restores a selected damaged record.
- English and Russian localization for the new flow.

## Next Pass

- First-person Pocket Codex model and open/close hand animation.
- Distinct content and behavior for Home, Signals, and Activity screens.
- Signal discovery and storage.
- Timed or interactive Decryptor mini-game instead of immediate restoration.
- Dock charging and battery consumption.
- Data-driven Codex entries loaded from JSON or datapacks.
- Automatic field records from first encounters and scripted events.
- Dedicated visual model and inserted-device state for the dock.
- Multiplayer validation and in-game visual QA.

## Explicitly Excluded

- Flash drive implementation and progression are not modified in this pass.
